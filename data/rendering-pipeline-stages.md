### 概要

渲染管线是将 3D 场景转换为 2D 图像的一系列处理步骤，可按职责拆成应用阶段、几何阶段、光栅化阶段。

### 内容

#### 三大阶段

| 阶段 | 主导 | 职责 | 输出 |
|------|------|------|------|
| 应用阶段 | CPU | 处理场景数据 | 渲染图元（点、线、面） |
| 几何阶段 | GPU | 处理几何体 | 屏幕空间顶点坐标、深度、着色信息 |
| 光栅化阶段 | GPU | 逐像素渲染 | 最终屏幕图像 |

#### 应用阶段

应用阶段由 CPU 主导，典型流程是加载数据、设置渲染状态、提交 Draw Call。

- 加载数据：Mesh、材质、贴图等从硬盘进入内存，再进入显存。
- 设置渲染状态：指定 Mesh、材质、贴图和管线状态的关联关系。
- 提交 Draw Call：CPU 调用图形 API，要求 GPU 开始绘制。

关键点：GPU 不能直接访问普通内存，渲染数据需要进入显存；视锥剔除、遮挡剔除等粗粒度剔除通常在应用阶段或其相邻系统中完成。

#### 几何阶段

几何阶段由 GPU 主导，典型链路是：

```text
Draw Call 图元 -> 顶点着色器 -> 曲面细分着色器 -> 几何着色器 -> 裁剪 -> 屏幕映射
```

并非所有项目都会启用曲面细分或几何着色器；基础理解重点是顶点被变换到裁剪空间，并准备后续阶段需要插值的数据。

#### 光栅化阶段

光栅化阶段由 GPU 主导，典型链路是：

```text
屏幕坐标 -> 三角形设置 -> 三角形遍历 -> 片元着色器 -> 逐片元操作 -> 目标缓冲区
```

片元阶段的成本与屏幕覆盖面积强相关，因此同一个模型在屏幕上越大，像素着色和后续逐片元操作通常越贵。

#### 记忆技巧

旧记录保留了“355”记忆法：

- 应用阶段 3 步骤：加载数据、设置渲染状态、提交 Draw Call。
- 几何阶段 5 步骤：从顶点着色器到屏幕映射。
- 光栅化阶段 5 步骤：从三角形设置到目标缓冲区。
